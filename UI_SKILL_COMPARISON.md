# UI 스킬 비교

## 대상 스킬

- `frontend-design`
- `ui-ux-pro-max`
- `web-design-guidelines`

## 한 줄 요약

- `frontend-design`: 고품질 UI를 실제로 구현하는 데 최적화된 제작 중심 스킬
- `ui-ux-pro-max`: 스타일/팔레트/타이포/UX 규칙을 데이터 기반으로 탐색·설계하는 스킬
- `web-design-guidelines`: 완성된 UI 코드를 가이드라인 기준으로 감사(audit)하는 스킬

## 목적별 차이

| 스킬 | 주 목적 | 가장 잘하는 일 | 산출물 |
|---|---|---|---|
| `frontend-design` | 제작 | 화면/컴포넌트 구현, 비주얼 완성도 향상 | 실제 UI 코드 |
| `ui-ux-pro-max` | 설계/선정 | 스타일·컬러·폰트·UX 규칙 선택, 설계 시스템 제안 | 설계 방향/디자인 시스템 |
| `web-design-guidelines` | 검수 | 접근성/UX/성능/폼/포커스 등 규정 준수 점검 | 리뷰 리포트(문제 목록) |

## 언제 쓰는지

### 1) `frontend-design`

- 랜딩 페이지, 대시보드, 컴포넌트를 빠르게 구현해야 할 때
- “UI를 더 세련되게 만들어줘” 같은 요청일 때
- 결과물로 바로 동작하는 코드가 필요할 때

### 2) `ui-ux-pro-max`

- 디자인 방향이 애매해서 스타일 결정을 먼저 해야 할 때
- 제품 성격에 맞는 색/타이포/레이아웃 기준을 정하고 싶을 때
- 구현 전에 일관된 디자인 시스템을 잡고 싶을 때

### 3) `web-design-guidelines`

- 구현 후 “기본 품질선”을 점검하고 싶을 때
- 접근성/포커스/폼/모션/성능 규칙 위반을 잡고 싶을 때
- 배포 전 UI 감사 체크를 하고 싶을 때

## 충돌 없이 같이 쓰는 방법 (권장 워크플로우)

1. `ui-ux-pro-max`로 디자인 기준(토큰/원칙) 먼저 확정
2. `frontend-design`으로 기준에 맞춰 구현
3. `web-design-guidelines`로 최종 감사

이 순서를 쓰면 “설계-구현-검수”가 분리되어 충돌을 줄일 수 있다.

## 핵심 결론

- 세 스킬은 중복이 아니라 역할 분담 관계다.
- `frontend-design`은 만든다.
- `ui-ux-pro-max`는 고른다/정한다.
- `web-design-guidelines`는 검사한다.
