# AI 협업 방법 / 룰 (재민용)

## 목적
AI(Codex)와 재민이 같은 기준으로 빠르게 작업/검토/재지시를 반복하기 위한 규칙 문서.

## 기본 흐름
1. AI: `AI_ORDERS_NOW.md` 갱신(오더 발행)
2. 재민: 오더 실행
3. 재민: `검토해줘` 신호
4. AI: 코드/문서/테스트 검증 + 판정 기록
5. AI: 다음 오더 갱신
6. 반복

## 스킬 사용 (고정)
- 스킬 경로: `ai_talk/jaemin-review-loop/SKILL.md`
- 트리거: `검토해줘`, `재민 작업 리뷰`, `추가 오더 줘`
- AI 응답 포맷: `Findings -> Orders -> Verify -> Report` 4블록 고정

## 역할 경계 (중요)
- 감독/우선순위 결정권: AI
- 실행/구현 책임: 재민
- 재민은 `AI_ORDERS_NOW.md` 기준으로만 작업
- 우선순위 변경은 제안 가능, 확정은 AI가 수행
- 결과의 최종 기록/판정 업데이트는 AI가 수행

## 커뮤니케이션 규칙
- 신호 문구는 `검토해줘`로 통일
- 애매한 말 대신 파일/기능 단위로 명시
- 완료 주장은 반드시 테스트/빌드 결과와 함께 기록
- 완료 보고 시 문서 기록 필수

## 재민 결과 기록 규칙
기록 파일: `ai_talk/JAEMIN_TASKS.md`

아래 항목을 매번 업데이트:
- `완료 항목`
- `변경 파일`
- `검증 결과` (`pnpm test`, `pnpm build`)
- `남은 이슈 / 리스크`
- `다음 제안 1~2개`

## AI 리뷰 출력 포맷 (고정)
AI는 항상 아래 4블록으로 답변:

### 1) Findings
- 심각도 순으로 문제 나열
- 각 항목에 파일 경로/라인 포함
- 문제 없으면 `중요 이슈 없음` 명시

### 2) Orders
- 재민이 즉시 실행할 작업 3~6개
- 각 작업은 완료 기준(DoD) 포함

### 3) Verify
- 실행할 검증 커맨드와 통과 기준
- 예: `pnpm test` (전부 PASS), `pnpm build` (error 0)

### 4) Report
- `JAEMIN_TASKS.md`에 반영할 문장/항목 지시

## 코드 작업 룰
- 기존 기능 깨는 변경 금지 (회귀 방지 우선)
- 임시 주석/죽은 코드 남기지 않기
- 추측 수정 금지: 근거 없는 수정은 테스트로 입증할 것
- 큰 변경은 작은 단위 커밋 가능 상태로 쪼개서 진행

## 품질 게이트 (머지 전)
아래 조건을 모두 만족해야 "완료"로 판단:
- `pnpm test` PASS
- `pnpm build` SUCCESS
- 관련 문서 업데이트 완료
- 새 기능에 대응하는 테스트 최소 1개 이상

## 긴급도 기준
- P0: 실행 불가/데이터 손상/핵심 기능 중단 → 즉시 수정
- P1: 기능 오동작/회귀 위험 높음 → 당일 수정
- P2: UX/구조 개선/리팩터링 → 스프린트 내 처리

## 예외 처리
- 재현 불가 버그: 재현 조건 먼저 문서화
- 범위 초과 작업: `남은 이슈 / 리스크`에 분리 기록
- 의존성 이슈: 임시 우회안 + 근본 해결안 둘 다 제시
